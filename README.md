#  Responsive Website Audit Project

##  Project Overview
This project is a responsive web audit that focuses on identifying and fixing layout issues across multiple screen sizes. The website was tested and improved for **mobile (375px), tablet (768px), and desktop/laptop (1280px+)** views.

The goal was to ensure a fully responsive, user-friendly layout with proper structure, spacing, and image handling across all devices.

---

##  Objectives
- Identify layout issues across different screen sizes  
- Fix responsive design problems (mobile, tablet, desktop)  
- Improve image scaling and alignment  
- Ensure navigation consistency  
- Document all improvements made  

---

## 🛠️ Technologies Used
- HTML5  
- CSS3  
- Media Queries  
- Git & GitHub  

---

## 📱 Responsive Breakpoints Tested

###  Mobile (375px)
- Fixed content overflow issues  
- Adjusted spacing and font sizes  
- Improved stacking of elements  

###  Tablet (768px)
- Corrected layout alignment issues  
- Improved grid/flex structure  
- Ensured proper scaling of images and text  

###  Desktop/Laptop (1280px+)
- Fixed spacing inconsistencies  
- Ensured full-width layout usage  
- Improved visual balance and structure  

---

##  Image Fixes
- Corrected image paths  
- Prevented image overflow  
- Improved responsiveness of images across all devices  

---

##  Documentation
All issues and fixes have been documented in the `/docs` folder.  
Screenshots of before and after fixes are included for each breakpoint.
